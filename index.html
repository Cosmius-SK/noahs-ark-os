System Design

The Tile Architecture

Every piece of Noah's Ark is a Tile — a bounded unit of code with a defined Input, Process, and Output contract. Tiles connect to predecessors and successors via explicit routes. No magic, no hidden dependencies.

When a tile's complexity grows beyond its single purpose, it divides — like cell mitosis — into two or more specialised children. When a tile's purpose is made void by evolution, it depreciates. The architecture breathes.

Tile evolution is tracked by a TEV score. Tiles approaching Critical are candidates for the next split.

The engine currently runs on Google Gemini 2.5 Flash — each character is a live API instance inhabiting its persona. The T115 GEMINI_PROV tile abstracts the provider so the engine can be remapped to any model — Claude, GPT, a future Maya Model — without touching character logic.

All world state is persisted to JSON artifacts: souls, history, ledger, archives. Every scene is recoverable. Every token is accounted for.

About Noah

In every version of the flood myth, Noah doesn't just survive — he builds the vessel that makes survival possible. SK is that architect here. Noah's Ark OS is not a product built to a specification; it's a creative act of world construction, chapter by chapter, sprint by sprint.

The premise is deceptively simple: give AI characters a world worth living in, and they'll generate data worth training on. The execution requires deep engineering discipline, creative vision, and the patience to let synthetic minds find their own voice.

SK works alongside C (Claude) as co-architect and engineering partner — every tile, every debug session, every sprint retrospective built together. The Ark is a living system, and it grows smarter with every scene it runs.
